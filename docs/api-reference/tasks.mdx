## Visão Geral

Os endpoints de tarefas permitem gerenciar tarefas associadas a projetos, incluindo criação, listagem, busca, atualização e exclusão.

<Note>
Todas as tarefas devem estar associadas a um projeto existente.
</Note>

## Modelo de Dados

### Task

```javascript
{
  id: INTEGER (PK, Auto Increment),
  title: STRING (NOT NULL),
  description: TEXT,
  status: ENUM('pending', 'in_progress', 'completed', 'cancelled') DEFAULT 'pending',
  priority: ENUM('low', 'medium', 'high', 'urgent') DEFAULT 'medium',
  dueDate: DATE,
  projectId: INTEGER (FK to projects.id),
  createdAt: DATE,
  updatedAt: DATE
}
```

### Campos Obrigatórios

- **title**: Título da tarefa (string, obrigatório)
- **projectId**: ID do projeto (integer, obrigatório)

### Campos Opcionais

- **description**: Descrição da tarefa (text)
- **status**: Status da tarefa (enum: 'pending', 'in_progress', 'completed', 'cancelled', padrão: 'pending')
- **priority**: Prioridade da tarefa (enum: 'low', 'medium', 'high', 'urgent', padrão: 'medium')
- **dueDate**: Data de vencimento (date)

## Exemplos de Uso

### Criar Tarefa

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/projects/1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Implementar autenticação",
    "description": "Desenvolver sistema de login e registro",
    "priority": "high",
    "dueDate": "2024-02-15"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/projects/1/tasks', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    title: 'Implementar autenticação',
    description: 'Desenvolver sistema de login e registro',
    priority: 'high',
    dueDate: '2024-02-15'
  })
});

const task = await response.json();
console.log(task);
```

```python Python
import requests

response = requests.post('http://localhost:3000/projects/1/tasks', json={
    'title': 'Implementar autenticação',
    'description': 'Desenvolver sistema de login e registro',
    'priority': 'high',
    'dueDate': '2024-02-15'
})

task = response.json()
print(task)
```

</CodeGroup>

## Validações

### Campos Obrigatórios

- **title**: Deve ser uma string não vazia
- **projectId**: Deve ser um ID de projeto existente

### Validações Específicas

- **status**: Deve ser 'pending', 'in_progress', 'completed' ou 'cancelled'
- **priority**: Deve ser 'low', 'medium', 'high' ou 'urgent'
- **dueDate**: Deve ser uma data válida no formato ISO 8601
- **title**: Máximo de 255 caracteres
- **description**: Máximo de 2000 caracteres

## Relacionamentos

<Note>
Todas as tarefas devem estar associadas a um projeto existente. Ao excluir um projeto, todas as suas tarefas também são excluídas.
</Note>

## Cache

<Note>
Todos os endpoints de tarefas utilizam cache Redis com TTL de 10 minutos. O cache é invalidado automaticamente quando dados são modificados.
</Note>
