## Visão Geral

Os endpoints de projetos permitem criar, listar, buscar, atualizar e excluir projetos no sistema LigueLead.

<Note>
Todos os endpoints retornam dados em formato JSON e seguem os padrões REST.
</Note>

## Endpoints Disponíveis

## Modelo de Dados

### Project

```javascript
{
  id: INTEGER (PK, Auto Increment),
  name: STRING (NOT NULL),
  description: STRING (NOT NULL),
  status: ENUM('active', 'inactive') DEFAULT 'active',
  createdAt: DATE,
  updatedAt: DATE
}
```

### Campos Obrigatórios

- **name**: Nome do projeto (string, obrigatório)
- **description**: Descrição do projeto (string, obrigatório)

### Campos Opcionais

- **status**: Status do projeto (enum: 'active' ou 'inactive', padrão: 'active')

## Exemplos de Uso

### Criar Projeto

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/projects \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sistema de Vendas",
    "description": "Desenvolvimento de sistema completo de vendas online"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/projects', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Sistema de Vendas',
    description: 'Desenvolvimento de sistema completo de vendas online'
  })
});

const project = await response.json();
console.log(project);
```

```python Python
import requests

response = requests.post('http://localhost:3000/projects', json={
    'name': 'Sistema de Vendas',
    'description': 'Desenvolvimento de sistema completo de vendas online'
})

project = response.json()
print(project)
```

</CodeGroup>

<CodeGroup>



## Validações

### Campos Obrigatórios

- **name**: Deve ser uma string não vazia
- **description**: Deve ser uma string não vazia

### Validações Específicas

- **status**: Deve ser 'active' ou 'inactive'
- **name**: Máximo de 255 caracteres
- **description**: Máximo de 1000 caracteres

## Cache

<Note>
Todos os endpoints de projetos utilizam cache Redis com TTL de 10 minutos. O cache é invalidado automaticamente quando dados são modificados.
</Note>
